# v0.1.54 - User/Global Configuration Support

**Release Date:** 2025-01-07

Adds user/global configuration file support for machine-specific settings like thermal management and Ollama host, separate from project-specific configuration.

---

## Added

- Add user/global configuration support (`~/.config/amanmcp/config.yaml`)
- Add `amanmcp config` command with `init`, `show`, and `path` subcommands
- Add embedded config templates in binary (via `//go:embed`)
- Add 4-tier configuration precedence: defaults → user → project → env

## Changed

- Update `amanmcp init` to show hint about user config for machine-specific settings
- Move project config template from hardcoded string to embedded file

## Documentation

- Update README.md with user config documentation and configuration precedence
- Update thermal-management.md to reference user config instead of env vars

---

## Upgrade Notes

**No action required for most users.** The defaults continue to work without any configuration.

For users who want to customize machine-specific settings (thermal management, Ollama host):

```bash
# Create user config from template
amanmcp config init

# Edit the file
# ~/.config/amanmcp/config.yaml
```

Configuration precedence (lowest to highest):
1. Defaults (hardcoded in binary)
2. User config (`~/.config/amanmcp/config.yaml`)
3. Project config (`.amanmcp.yaml`)
4. Environment variables (`AMANMCP_*`)

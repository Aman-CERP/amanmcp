# v0.6.0

**Release Date:** 2026-01-14

---

## Added

- **SQLite FTS5 BM25 backend** (REARCH-002): Replace Bleve/BoltDB with SQLite FTS5 for concurrent multi-process access. Fixes BUG-064 (CLI blocked when MCP server running). SQLite with WAL mode enables multiple readers + single writer.
- **BM25 backend factory**: `store.NewBM25IndexWithBackend()` factory for selecting between sqlite (default) and bleve backends
- **Backend auto-detection**: Factory auto-detects existing index backend for backward compatibility
- **`--local` flag for search**: Force local search, bypassing daemon (useful for testing)

## Changed

- **Default BM25 backend is now SQLite**: New indexes use SQLite FTS5 instead of Bleve for better concurrent access
- **Config option `bm25_backend`**: Add `search.bm25_backend` config option to explicitly select "sqlite" or "bleve"

## Fixed

- **BUG-064**: CLI search/validation blocked when MCP server is running (BoltDB exclusive file lock issue). SQLite FTS5 with WAL mode enables concurrent access.

# v0.9.0 - Zero-Config UX

**Release Date:** 2026-01-15

This release delivers the "Zero-Config UX" vision - true 3-command setup for amanmcp. Ollama lifecycle is now fully automated, eliminating the need for manual embedding backend management.

---

## Added

- Zero-config Ollama lifecycle management in `amanmcp init` (DEBT-026)
  - Auto-detects Ollama installation (CLI and .app on macOS)
  - Auto-starts Ollama if installed but not running
  - Auto-pulls embedding model if missing with progress bar
  - Interactive prompts for non-installed case (TTY only)
  - Clear error messages with install instructions for CI/scripts

- New `internal/lifecycle` package for Ollama lifecycle management
  - `OllamaManager` with detection, startup, and model pulling
  - Platform-specific startup (macOS: `open -a Ollama`, Linux: systemd/direct)
  - `IsTTY()` detection and interactive prompts
  - Progress bar for model downloads

- Enhanced `amanmcp setup` command
  - `--check` flag: Status-only mode
  - `--auto` flag: Non-interactive mode for scripts/Homebrew
  - `--offline` flag: Configure for BM25-only search
  - Shows Ollama installation, running, and model status

- Add `--explain` flag to `amanmcp search` for search decision transparency (FEAT-UNIX3)
  - Shows BM25/Vector result counts and weights
  - Displays RRF fusion constant (k value)
  - Reports per-result BM25 rank, vector rank, and scores
  - Indicates search mode (hybrid, BM25-only, multi-query decomposition)
  - Implements Unix Rule of Transparency for debugging

## Fixed

- Fix silent fallback to static embeddings when Ollama unavailable (BUG-073)
  - Index command now fails with clear error message instead of silently using useless static embeddings
  - Error includes actionable fix: "Start Ollama: ollama serve" or "Use BM25-only: --backend=static"
  - Search command skips embedder creation when `--bm25-only` is specified

## Documentation

- Simplify README Quick Start to 3 commands (DEBT-026)
  - Remove manual Ollama setup steps
  - Document automatic Ollama lifecycle management
  - Add "What `amanmcp init` Does Automatically" section
  - Update `amanmcp setup` command documentation

## Quick Start (Now Just 3 Commands!)

```bash
# 1. Install AmanMCP
brew tap Aman-CERP/tap && brew install amanmcp

# 2. Initialize your project (auto-starts Ollama, pulls model if needed)
cd /path/to/your/project
amanmcp init

# 3. Restart Claude Code to activate
```
